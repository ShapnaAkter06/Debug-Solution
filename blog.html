Local Storage:

⇒ Stores data in browser without any expiration date.
⇒ Save Data in key/value pairs in the browser.
⇒ Data is not deleted when browser is closed.
⇒ Data is available for future session.

Session storage:

⇒ Stores data in browser for a session.
⇒ Save Data in key/value pairs in the browser.
⇒ Data is deleted when browser is closed.
⇒ Data is not available for future session.

Global scope

The variables defined outside of any function or curly brackets are known as global variables and have global scope. Global scope means that the variables can be accessed from any part of that program, any function or conditional state can access that variable.

Block scope

Block scope is also a sub-type of local scope. The block scope can be defined as the scope of the variables inside the curly brackets {}. Now, these curly brackets can be of loops, or conditional statements, or something else. It's allowed to refer to these variables from within the curly brackets {}.

There are 8 ways to get undefined

1. variable that is not initialized will give undefined.
2. function with no return.
3. Parameter that is not passed will be undefined.
4. If return has nothing on the right side will return undefined.
5. If property does not exists on an objects.
6. accessing array elements outside of the index range.
7. deleting an array element inside an array.
8. set a value directly to undefined. example: const age = undefined.
